User model
id, name, username, email, password, is_admin, timestamps
contoh role: USR || ADM --> 0 || 1 , 1 = admin
USR (user) -> bisa add mahasiswa dan process
ADM (admin) -> bisa crud kriteria, semuanya

Student model
id, nama, nim, alamat, ipk, ips, pendapatan_ortu, jumlah_saudara, biaya_hidup, foto, logo_univ, timestamps

Criteria model
id, nama_kriteria, bobot, tipe, timestamps
contoh tipe: beneficial (1) || non beneficial (0), bisa pakai boolean

Category model
id, nama_kategori
contoh nama kategori: Mahasiswa UMN Angkatan ... / Semester ...

Result model
id, peringkat, nim, nama, skor, timestamps

https://laravel.com/docs/8.x/eloquent-relationships#many-to-many
category_student table

Kriteria
ipk             benefit
ips             benefit
pendapatan_ortu cost
jumlah_saudara  benefit
biaya_hidup     benefit

php artisan make:controller LoginController
php artisan make:controller RegisterController

php artisan make:controller ControllerName --resource --model=ModelName
php artisan make:controller CriteriaController --resource --model=Criteria

php artisan migrate

Run seeder:
php artisan db:seed

Run migrate:fresh + seeding:
php artisan migrate:fresh --seed
